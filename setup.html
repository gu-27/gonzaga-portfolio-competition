Tech Setup

From laptop to live portfolio.

Zero experience required. This page walks you through every step — GitHub, VS Code, and publishing your site to the internet.

If you've never used GitHub or written code before, that's fine. This page assumes zero experience. The setup will take a few hours — maybe longer — and it will involve frustration. That frustration is part of the point.

Every student who completes the setup walks away with a story they can tell in interviews: "I was given general direction, no template, and I figured it out." Employers love that story. Embrace the struggle.

If you get stuck, that's normal. Read the troubleshooting section at the bottom, ask Claude Code for help, or search the web. Working through a problem you've never seen before is the skill. You're already practicing it.

Before You Start

What You'll Need

  • A laptop — Mac or Windows both work fine
  • An email address you check regularly
  • 2–4 hours of uninterrupted time (you can break this into multiple sessions)
  • A willingness to search for answers when something doesn't work
💡 Note

You do not need to know how to code. You will learn just enough to get your portfolio live — and more importantly, you'll understand why each step exists.

The Setup

Six Steps to Live

1

Create a GitHub Account

⏱ 10–15 minutes

What GitHub is: GitHub is a platform where developers store, share, and collaborate on code. Think of it as Google Drive, but for code — except the files are version-controlled (meaning you can see every change ever made) and publicly visible.

When you publish a portfolio through GitHub Pages (Step 5), anyone in the world with your URL can see your work. That's what makes it a real artifact.

How to sign up:

  1. Go to github.com
  2. Click Sign up and enter your email address
  3. Choose a username — this becomes part of your portfolio URL, so pick something professional. firstname-lastname or firstnamelastname works well.
  4. Verify your email address
🎯 Why this matters

Recruiters look at GitHub profiles. The earlier you start building one, the more history you have when it matters. Your GitHub profile from this summer is infrastructure for the rest of your career.

2

Install Visual Studio Code

⏱ 10 minutes

What it is: VS Code is a free code editor made by Microsoft. It's what most developers use to write and edit files. You'll use it to write HTML and CSS for your portfolio.

  1. Go to code.visualstudio.com
  2. Click the download button for your operating system (Mac or Windows)
  3. Install it like any other application — drag to Applications on Mac, run the installer on Windows
  4. Open VS Code and take a few minutes to look around. Don't worry about understanding everything — you'll learn as you go.

Basic orientation:

  • Left sidebar: your file explorer — this is where you open folders and see your files
  • Main area: your editor — this is where you write code
  • Bottom bar: terminal and status info — you'll use the terminal in later steps
💡 Tip

Press Ctrl + ` (Windows) or Cmd + ` (Mac) to open a terminal directly inside VS Code. You'll use this a lot.

3

Install Claude Code (Optional but Recommended)

⏱ 15–30 minutes

What it is: Claude Code is an AI assistant that runs directly inside VS Code. Unlike a chat window, it can read your actual files, write code with you, explain errors, and help you think through decisions — all while staying in context with what you're actually building.

You are allowed — and encouraged — to use AI tools in this competition. Claude Code is the recommended option because it's the most useful for a project like this.

⚡ Why it's a superpower

Claude Code doesn't replace your thinking — it amplifies it. If you bring clear direction and real decisions, it will execute faster and better than you could alone. If you bring nothing, it produces nothing useful. Your judgment is still the product.

Installation instructions and setup guide: Claude Code Setup Guide → (Link coming soon — check back here before you start)

4

Create Your Portfolio Repository

⏱ 10 minutes

What a repository is: A repository (or "repo") is a folder on GitHub that contains all your project files. Every change you make is tracked. Think of it as a save file for your entire project — except it saves every version, forever.

  1. Log into GitHub at github.com
  2. Click the + icon in the top-right corner, then New repository
  3. Name your repository. For GitHub Pages to work cleanly, name it: 
    your-username.github.io
    Replace your-username with your actual GitHub username, exactly.
  4. Set visibility to Public — required for free GitHub Pages hosting
  5. Check Add a README file
  6. Click Create repository
⚡ The naming convention matters

If your username is jsmith, your repo should be named jsmith.github.io — exactly. This is the magic incantation that tells GitHub to serve it as a personal site at that URL.

5

Enable GitHub Pages

⏱ 5 minutes

GitHub Pages is a free hosting service built into GitHub. Once enabled, it automatically publishes your repository as a live website at your-username.github.io — no server setup required.

  1. In your new repository, click the Settings tab (near the top)
  2. In the left sidebar, scroll down and click Pages
  3. Under Branch, select main and click Save
  4. Wait about 60–90 seconds, then reload the Settings → Pages view
  5. You'll see a banner: "Your site is live at https://your-username.github.io"
💡 Note

The site will show just your README for now — and it won't look like much. That's fine. You've done something real: you have a live URL. Everything from here is making that URL worth visiting.

6

Your First Commit

⏱ 20–30 minutes

A commit is a snapshot of your files at a moment in time. Every time you "push" changes to GitHub, you're creating a commit with a message describing what you changed. It's like a save point with a note attached.

Clone your repository to your laptop:

In VS Code, open the terminal (Cmd + ` on Mac, Ctrl + ` on Windows), then run: 

# Replace YOUR-USERNAME with your actual GitHub username
git clone https://github.com/YOUR-USERNAME/YOUR-USERNAME.github.io
cd YOUR-USERNAME.github.io

Create your first file:

In VS Code, create a file called index.html in that folder and add some content. Even a simple placeholder works: 

<!DOCTYPE html>
<html>
<head>
  <title>My Portfolio</title>
</head>
<body>
  <h1>Coming soon.</h1>
</body>
</html>

Push it to GitHub:

git add index.html
git commit -m "First commit — hello world"
git push
🎯 What just happened

You created a file, staged it (git add), labeled the snapshot (git commit), and sent it to GitHub (git push). Within 60–90 seconds, your live URL will show your new file. Refresh https://your-username.github.io and see for yourself.

This is the workflow you'll repeat hundreds of times this summer. It gets fast.

When Things Break

Troubleshooting

Things will go wrong. Here's how to handle it.

git push is asking for a username and password +

GitHub deprecated password authentication over HTTPS. You need to set up a Personal Access Token (PAT) or use SSH instead.

Easiest fix: go to GitHub → Settings → Developer settings → Personal access tokens → Generate new token. Give it repo permissions. Use this token as your "password" when git asks. Store it somewhere safe — you can't see it again after creation.

Even easier: ask Claude Code to walk you through setting up SSH authentication. Once done, you'll never deal with this again.

My GitHub Pages site is showing a 404 +

Two likely causes:

  • Pages isn't enabled yet — double-check Settings → Pages and that the branch is set to main
  • No index.html file — GitHub Pages looks for a file named exactly index.html in the root of your repo. If it doesn't exist, you get a 404.

Wait 2–3 minutes after any change and force-refresh your browser (Cmd + Shift + R on Mac).

git commit says "Author identity unknown" +

Git doesn't know who you are yet. Tell it:

git config --global user.email "you@example.com"
git config --global user.name "Your Name"

Use the same email address as your GitHub account. Then run your commit again.

I got an error I don't understand +

Copy the exact error message. Then:

  1. Ask Claude Code — paste the error and say "I was doing X and got this error." It will explain what went wrong and how to fix it.
  2. Search the exact error text on Google — you are almost certainly not the first person to hit this. GitHub Docs and Stack Overflow are excellent resources.
  3. GitHub Docs at docs.github.com — comprehensive, well-written, and has a good search.

The process of diagnosing and fixing an error you've never seen before is exactly what employers pay people to do. You're doing real work right now.

I have no idea if I'm doing this right +

Here's the benchmark: does your URL load a page? If yes, you're doing it right. Everything else is refinement.

The steps in this guide are the exact steps used by professionals. If you've followed them and your site is live, you've done something real — not a simulation of something real.
